# Document Fragmentation for Token Efficiency

## The Idea
Technical optimization: Create sub-folders or fragment documents when they grow too large to avoid violating the rule of token consumption optimization.

For example, reading a full Code Map in its entirety when working on a task that only involves a small portion of it doesn't make sense and wastes tokens.

## Goal
Establish a pattern for splitting large documents (especially those that tend to grow continuously) into smaller, more focused fragments. This might not be necessary for all documents, but specifically for those prone to bloating.
